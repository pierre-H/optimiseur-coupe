Cut optimizer



Instruction manual



Summary





Goal of this software



The user has a certain amount of bars that can have different dimensions, and wants to cut them in of different section length.

This program helps defining which cut combination will be the best, (which combination will have the biggest section remaining).



Langage choice



The software detects when starting which langage is natively used by OS. If the french langage is detected, it will then choose the french. If not, if will choose english.



Global overview of the user's interface.



The interface has 3 tabs  :



- One tab is dedicated to key-in :



- One tab where the results will be displayed as text :



- One tab where the results will be displayed as graphic / charts :





TAB « Data entry Form »



There are 3 subsections, one for the bar length before cutting, one for the wished sections dimensions, the last one to choose the thickness of the blade.

Once the data entered, the button « apply » will launch the calculation :



Clicking on the « help » button will open this user guide :



By clicking on the button « QUIT », the user will exit from the application :



Collecting the length before cutting



Each data entry line has 3 datas fields :



- Lenght :



A spinbox enables the user to choose either a manual entry or increase or decrease the value displayed in the field. Thanks this, it is possible to choose the length of one or several bars.



- Unit of measure :



A drop down list enables the user to choose among 6 bar unit of measure : mm, cm, m, inch, foot and yard.



- Quantity :



A spinbox enables to choose either a manual data entry, or to increase or decrease the default value from the field.



Eventually, clicking on this button will create a new line to enter data :



Another button is here to delete the last line entered :



Enter the wished length of the section





Each line has 3 fields where data can be entered :



- Length:



A spinbox enables to choose either to enter the data manually or to increase or decrease the default value of the field. The user can therefore choose the length of one or several sections.



- Unit of measure :

Thanks to a drop down list the user can choose between 6 different section unit of measures  : mm, cm, m, inch, foot and the yard.



- Quantity :



A spinbox enables the user to choose either to enter the data manually or to increase or decrease the default value of this field.



Eventually, clicking on this button will create a new line to enter data :



This other button will delete the last line :



Enter the thickness of the blade





This tab has only 2 fields:



- Thickness of the blade  :



A spinbox enables the user to choose either to enter the data manually or to increase or decrease the default value of this field.



- Unit of measure :



A drop down list enable to choose between two thickness unit of measures : mm and the inch.



Tab « results »



The results can either be shown like text or graphically.



The text :



- The list of the cut, for each bar, in the native unity of measure entered

- The yield on each raw material bar

- The average yield



Just like in the tab « data enter tab », the user will find a button « Help » to reach this user guide, as well as a button « Quit », to exit from the application .



The cutting drawing results :



For each raw material bar, there will be a drawing for the cutting, showing the size of the section to cut.

Just like in the tab « data enter tab », the user will find a button « Help » to reach this user guide, as well as a button « Quit », to exit from the application .



Error pop ups



Raw material shortage



The following error message will pop up when there will not be enough raw material bar length to cut the required cuts quantity.



Raw material to short



This error message will pop up when no raw material bar is long enough to make the required cut .



Wrong yield causing raw material shortage :



This error message will pop up if, inspite of a quantity looking sufficient, wrong yields are preventing from getting the required sections.
